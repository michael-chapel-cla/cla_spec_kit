# Plans Directory

This directory contains the implementation plans generated by the `/plan` command.

## Structure

```
plans/
└── <app-name>/
    └── PLAN.md       ← The primary build plan (architecture, endpoints, DB schema, file structure)
```

## How plans are generated

Plans are created by running:

```
/plan <app-name>
```

The command reads all files in `/requirements/<app-name>/` and produces a detailed technical plan covering:

- Architecture decisions
- Complete directory and file structure for backend, frontend, and helm
- Every API endpoint with method, path, auth requirements, and types
- Full database schema with Flyway migration file names
- Frontend pages and the API calls they make
- Helm chart configuration changes required for the app
- Azure Entra app registration requirements
- Local development setup steps
- Phased build roadmap (MVP → expansion → scale)

## How plans are used

The `/create <app-name>` command reads the plan and uses it to scaffold the application into `/repos/<app-name>/`. The plan is the source of truth for what gets built — if something is not in the plan, it will not be in the generated code.

If you need to change what gets built, edit `PLAN.md` before running `/create`.
