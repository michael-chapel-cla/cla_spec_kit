# Repos Directory

This directory contains the applications generated by the `/create` command.

## Structure

```
repos/
└── <app-name>/
    ├── backend/      ← Node.js + Fastify API (TypeScript)
    ├── frontend/     ← React + Vite SPA (TypeScript + MUI)
    └── helm/         ← Kubernetes deployment (Helm + AKS)
```

## How apps are generated

Applications are scaffolded by running:

```
/create <app-name>
```

The command reads `/plans/<app-name>/PLAN.md` and the standards from `/specs/`, then copies and customizes the starter templates to produce a working application scaffold.

## After generation

Each generated app in `repos/<app-name>/` is an independent repository candidate. The intended workflow after generation:

1. Review the scaffold and the `PLAN.md` it was built from
2. Create a new git repository for the app (the generated code is not committed to this spec-kit repo)
3. Continue development using GitHub Copilot with the context from `.github/copilot-instructions.md`

## Standards compliance

All generated apps are scaffolded to comply with the standards in `/specs/`. GitHub Copilot uses `.github/copilot-instructions.md` to apply these same standards when assisting with further development.
