# Repos Directory

This directory contains the applications generated by the `/create` command.

## Structure

Each application is split into three independent repositories, generated as sibling directories here:

```
repos/
├── web-api-<app-name>/    ← Fastify API — Node.js 20 + TypeScript
├── web-<app-name>/        ← React SPA — Vite + MUI v6 + TypeScript
└── db-<app-name>/         ← Flyway SQL migrations — MSSQL
```

Each repo is self-contained:
- Its own `docker-compose.yml`
- Its own `.devcontainer/devcontainer.json`
- Its own `azure-pipelines.yml`
- Its own `.github/copilot-instructions.md`
- API and frontend each have their own `helm/` chart

The API repo's `docker-compose.yml` mounts `../db-<app-name>/migrations` for Flyway, so all three repos should be cloned as siblings in production use.

## How apps are generated

Applications are scaffolded by running:

```
/create <app-name>
```

The command reads `/plans/<app-name>/PLAN.md` and the standards embedded in the prompt, then copies and customizes the starter templates to produce three working repository scaffolds.

## After generation

Each generated repo is an independent repository candidate. The intended workflow:

1. Review the scaffold and the `PLAN.md` it was built from
2. Create separate git repositories for each of the three repos (they are not committed to this spec-kit repo)
3. Continue development using GitHub Copilot — each repo carries its own `.github/copilot-instructions.md` with the full coding standards

## Local development (all three repos together)

Clone all three as siblings, then use the API repo's docker-compose for the database and backend, and the frontend repo's docker-compose for the SPA:

```bash
git clone <api-repo-url>      web-api-<app-name>
git clone <frontend-repo-url> web-<app-name>
git clone <db-repo-url>       db-<app-name>

cd web-api-<app-name>
cp .env.example .env           # edit with real values
docker compose up              # starts db + migrations + backend
```

```bash
cd web-<app-name>
docker compose up              # starts frontend on port 3000
```
